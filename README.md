# Учёт грибов

Веб-приложение для учёта производства и продаж на малом предприятии по переработке грибной продукции. Замена бумажного журнала. Локальная установка, один пользователь, телефон как основное устройство.

## Что умеет

- **Приёмка** — ежедневная запись по трём сортам (1, 2, 3) с автосуммой ₽, привязкой к поставщику и истории цен.
- **Сушка** — загрузка сырья, выход сухого по сортам с авторасчётом % выхода, раздельные расходы (свет, вода, дрова, зарплата сушильщикам), валидация остатков.
- **Мусор / отходы** — ежедневная запись с % отходов, подсветка дней с >25%.
- **Продажи** — реализация сухого гриба с проверкой остатка (нельзя продать больше, чем есть), привязка к покупателю.
- **Расходы (общие)** — аренда, транспорт, зарплата, прочее с произвольной категорией.
- **Справочники** — закупочные цены (с историей `effective_from`), поставщики (имя, телефон, заметка), покупатели.
- **Сезоны** — создание нового, мягкое закрытие старого, переключение активного, переименование, сводка по каждому.
- **Отчёты** — P&L, по поставщикам, по покупателям, по дням; с фильтром по периоду (сезон / сегодня / вчера / неделя / месяц / произвольный). Экспорт в CSV (`.csv` с BOM, открывается в Excel). Печать через Ctrl+P.
- **Главная** — остатки сырья и сухого, итоги дня, прибыль за сезон, онбординг-подсказки для пустой базы.

## Стек

- **Backend**: Python 3 + Flask
- **БД**: SQLite (файл `mushroom.db` создаётся автоматически)
- **Frontend**: серверный рендеринг Jinja2 + Tailwind CSS (CDN) + HTMX (CDN)
- **Без сборки**, без npm, без миграций, без интернета после первой загрузки страниц

## Запуск (локально)

```bash
# 1. Создать виртуальное окружение (один раз)
python -m venv venv

# 2. Активировать его
# Windows (PowerShell):
.\venv\Scripts\Activate.ps1
# Windows (cmd):
.\venv\Scripts\activate.bat
# Linux/macOS:
source venv/bin/activate

# 3. Установить зависимости (один раз)
pip install -r requirements.txt

# 4. Запустить
python app.py
```

После запуска:

- На компьютере, где запущен скрипт: `http://127.0.0.1:5000`
- На телефоне в той же Wi-Fi-сети: `http://<IP-этого-ПК>:5000`
  - Узнать IP:
    - Windows: `ipconfig` (ищите IPv4)
    - Linux/macOS: `ipconfig` или `ip a`

## Где хранятся данные

Один файл — `mushroom.db` в корне проекта. Чтобы сделать резервную копию — просто скопируйте этот файл. Чтобы перенести на другой компьютер — скопируйте всю папку `mushroom/`.

## Этапы разработки

- [x] **Этап 1**: каркас, авторизация, навигация
- [x] **Этап 2**: справочники (закупочные цены, поставщики) + приёмка с автосуммой
- [x] **Этап 3**: сушка (с % выхода по сортам) + мусор (с % отходов)
- [x] **Этап 4**: продажи (с проверкой остатка сухого) + общие расходы + справочник покупателей
- [x] **Этап 5**: сезоны — создание, мягкое закрытие, переключение, переименование, сводка по каждому
- [x] **Этап 6**: отчёты — P&L, по поставщикам, по покупателям, по дням; фильтр по периоду
- [x] **Этап 7**: полировка — онбординг, CSV-экспорт, кастомные 404/500, печатные стили, favicon

## Известные ограничения

- **Один пользователь**. Авторизация одиночная, ролей нет — система для одного владельца.
- **Нет редактирования записей**. Любую запись (приёмку, сушку, продажу, расход) можно только удалить и ввести заново. Это сознательное упрощение.
- **Один вид гриба** на все сезоны.
- **Без онлайн-синхронизации** — данные только на устройстве, где запущен сервер. Для мульти-устройства нужен общий Wi-Fi и запуск сервера на одном из них.
