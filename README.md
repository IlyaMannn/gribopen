# Учёт грибов

Веб-приложение для учёта производства и продаж на малом предприятии по переработке грибной продукции.

## Стек

- **Backend**: Python 3 + Flask
- **БД**: SQLite (файл `mushroom.db` создаётся автоматически)
- **Frontend**: серверный рендеринг Jinja2 + Tailwind CSS (CDN) + HTMX (CDN)
- **Без сборки**, без npm, без миграций

## Запуск (локально)

```bash
# 1. Создать виртуальное окружение (один раз)
python -m venv venv

# 2. Активировать его
# Windows (PowerShell):
.\venv\Scripts\Activate.ps1
# Windows (cmd):
.\venv\Scripts\activate.bat
# Linux/macOS:
source venv/bin/activate

# 3. Установить зависимости (один раз)
pip install -r requirements.txt

# 4. Запустить
python app.py
```

После запуска:

- На компьютере, где запущен скрипт: откройте `http://127.0.0.1:5000`
- На телефоне в той же Wi-Fi-сети: `http://<IP-этого-ПК>:5000`
  - Узнать IP:
    - Windows: `ipconfig` (ищите IPv4)
    - Linux/macOS: `ifconfig` или `ip a`

## Логин по умолчанию

- **Логин**: `admin`
- **Пароль**: `admin`

Сменить можно в меню пользователя (правый верхний угол) → «Настройки» → «Сменить пароль».

## Где хранятся данные

Один файл — `mushroom.db` в корне проекта. Чтобы сделать резервную копию — просто скопируйте этот файл. Чтобы перенести на другой компьютер — скопируйте всю папку `mushroom/`.

## Структура

```
mushroom/
├── app.py              # Flask-приложение
├── db.py               # подключение к SQLite, схема БД
├── auth.py             # авторизация
├── seed.py             # начальные данные (admin, 3 сорта)
├── mushroom.db         # создаётся автоматически
├── requirements.txt
├── templates/          # HTML-шаблоны Jinja2
├── static/             # CSS, иконки
└── README.md
```
